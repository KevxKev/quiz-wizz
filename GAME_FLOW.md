# Game Flow

## 1. Host starts a room
- Open the host page on the laptop connected to the TV.
- Create a room and display the room code on screen.
- Optionally show a QR code later for faster joining.

## 2. Players join on phones
- Players open the join page.
- They enter the room code and a display name.
- The host sees the player list update in real time.

## 3. Hidden quiz entries exist before play
- Friends can use the submission page before the party or during setup.
- They provide a YouTube link, artist, timing, and playback mode.
- These entries stay hidden from the host until selected for a round.

## 4. Host starts a round
- The host chooses a pending submission.
- The app launches the YouTube clip at the configured start time.
- The clip stops or locks at the configured end time.

## 5. Players answer privately
- Phones switch to the answer page for the active round.
- Each player submits a guess for the artist.
- Answers stay hidden until reveal.

## 6. Host reveals results
- The host reveals the correct artist.
- Submitted answers are shown.
- Correct players are marked.
- Scores update.

## 7. Repeat until game end
- The host advances to the next round.
- After the last round, the app shows the final leaderboard.
- A simple winner screen ends the session.

## MVP UX rules
- Keep player actions obvious and fast.
- Avoid too many buttons on the phone UI.
- Make the host screen readable from across a room.
- Keep answers hidden until the reveal moment.
